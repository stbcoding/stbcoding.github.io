Split testing page load speeds

This split tests refers to various setups with CloudFlare, Apache, Litespeed, Litespeed Cache Plugin and WP Rocket

Why I made this test:

After recommending LiteSpeed and Cloudflare with proxied CDN subdomain to a lot of people and after implementing it into a LOT of WordPress websites (recommendations based on my experience) I have decided to do a couple of split tests with different setups to have something to fully back up my words.

What is the server setup:

Hardware:
Lightsail $40 Server Placed in US East with 8 GB Memory 2 Core Processor 160 GB SSD Disk
Software Installed:
-WHM with 3 accounts
-Apache (out of the box, zero tweaks)
-Litespeed (out of the box, zero tweaks)
-PHP 7.3
-LetsEncrypt AutoSSL plugin
-MariaDB 10.3

How were the tests performed:

Web Server:
The web server was switched from LiteSpeed to Apache and viceversa using "Switch to..." functionality in the WHM Plugin.
All tests involving Cloudflare proxy-ing were done after a first load to "warm the cache".
This test was done on three distinct domains with the .xyz TLD.
Plugins:
The plugins were enabled or disabled from the WP plugins menu while clearing cache before disabling
Plugins used:
CDN Enabler was used with WP Rocket because WP Rocket did not manage to replace the pictures URLs from inside an Oxygen based webpage.
-Oxygen Builder (so the setup will involve a fast website)
-WooCommerce (to add some dynamic content and some pictures on the front page)
-Elegant Custom Fonts (to load fonts locally and it works well with Oxygen Builder)
-LiteSpeed Cache 3.0.9
-WP Rocket 3.5.4
-CDN Enabler 1.0.9

Why was WP Rocket included in this test:

Tap to Enlarge
This happened because I got served a Facebook Ad claiming they are the best and that 80% of the functionality was already enabled.
So I tested both LiteSpeed Cache and WP Rocket by simply enabling them and the only tweak was to set up CDN serving in the plugin settings.
Because I really dislike bold claims about "magic tools" like this I decided to include it into the test.
The website will load faster indeed but only by 5-10% with the basic "80% of web performace..." which is not a lot and free plugins can do the same stuff or even .htaccess edits.

Conclusions:

#1: LiteSpeed rocks!
I already knew this because I have set it up in a lot of ways before and on a lot of servers and it's just really fast especially vs Apache. I recommend it to all my clients.
#2: WP Rocket is not as good advertised.
After performing the tests I saw that on regular Apache installs WP Rocket is (somewhat) good but I would never pay for it. The money for WP Rocket would rather go into a better hosting. It has a lot of hype and probably similar (or slightly better or slightly worse) results are shared with all the "make WordPress fast" plugins.
#3: Cloudflare with just CDN subdomain proxy is just great.
There is a lot of bad-mouthing Cloudflare but with this setup is decent enough to call it a good alternative. Please see screenshot of correct settings using cPanel and Cloudflare (click to enlarge)
